관리자 비밀번호 변경

현재 로그인한 계정의 비밀번호를 변경합니다.

Firebase를 켜기 전에는 정적 호스팅용 브라우저 저장(localStorage) 방식으로 동작합니다. Firebase를 켠 뒤 비밀번호 변경은 Firebase Authentication 콘솔의 사용자 관리 또는 비밀번호 재설정 메일 기능을 사용해 주세요.

게시판 공개/교사별 열람 설정

공개를 체크한 자료는 로그인 없이 열리고, 교사 허용을 체크한 자료는 선택한 등록 교사에게만 열립니다. 게시판 제목 줄의 체크박스를 누르면 하위 메뉴가 한 번에 선택됩니다.

Firebase가 연결되면 저장한 공개/교사별 권한 설정이 Firestore에 저장되어 모든 방문자에게 같은 기준으로 적용됩니다. Firebase를 아직 켜지 않은 동안에는 현재 브라우저 또는 GitHub Pages 자동 반영 방식으로 동작합니다.

공지사항 팝업 설정

메인·자료 페이지에 띄울 공지 팝업을 설정합니다. “오늘 하루 열지 않기”, “다시 보지 않기”, 링크 버튼을 공지별로 설정할 수 있습니다.

Firebase Auth · Firestore · Storage 운영 구조

로그인은 Firebase Auth, 게시판 권한/공지 설정은 Firestore, 공지 이미지는 Storage에 저장하는 운영형 구조입니다.

Firebase 연결 상태를 확인하는 중입니다.
assets/js/firebase-config.js에 Firebase Web App 설정값을 넣고 enabled: true로 바꾼 뒤, firebase/firestore.rules와 firebase/storage.rules를 Firebase 콘솔에 배포하면 활성화됩니다. Firebase가 활성화되면 이 화면의 “설정 저장”, “공지 저장”은 Firestore/Storage에 바로 반영됩니다.

GitHub Pages 자동 반영

Firebase를 쓰지 않는 경우의 예비 방식입니다. 체크하면 게시판 공개 설정과 공지 설정을 GitHub 저장소의 assets/js/site-data.js에 커밋합니다.

보안을 위해 토큰은 저장하지 않습니다. GitHub 토큰은 Contents: Read and write 권한만 있는 Fine-grained token을 권장합니다. 토큰을 입력하지 않으면 설정은 현재 브라우저에만 저장됩니다.
